## 规则格式

__代码_块_0__

|职位|名称 |描述 |
|---|---|---|
| 1 |更换|规则匹配后的输出文本 |
| 2 |图案|要匹配的正则表达式或模糊字符串 |
| 3 |阈值|忽略正则表达式规则。用于模糊匹配 (0–100) |
| 4 |旗帜| `{'flags': re.IGNORECASE}` 表示不区分大小写，`0` 表示区分大小写 |

## 管道逻辑

- 规则的处理**自上而下**
- **所有**匹配规则均适用（累积）
- **完全匹配** (`^...$`) 立即停止管道
- 较早的规则优先于较晚的规则

## 你的第一条规则——一步一步

1. 打开 `config/maps/plugins/sandbox/de-DE/FUZZY_MAP_pre.py`
2. 在 `FUZZY_MAP_pre = [...]` 中添加您的规则
3. 保存 — Aura 自动重新加载，无需重启
4. 口述你的触发短语并观看它的触发


## 推荐的文件结构

将您的规则**放在**长评论块之前：
__代码_块_7__

**为什么？** Aura 的自动修复仅扫描文件的前 ~1KB。
如果您的规则出现在长标头之后，自动修复功能将无法找到或修复它们。
还推荐第 1 行的路径注释——它可以帮助人们快速识别文件。
