# Die License erklärt in deutscher Sprache

**Licencja Apache, wersja 2.0** jest ostatecznym **wolnym oprogramowaniem** (wolnym oprogramowaniem) i **oprogramowaniem typu open source**.

Hier sind die Szczegóły, warum das so ist und was das für dich bedeutet:

### 1. Oficjalne Anerkennung
* **FSF (Fundacja Wolnego Oprogramowania):** Sie stuft die Apache 2.0 als Lizenz für „Freie Software” ein. Sie ist allerdings **nicht kompatybilny mit der GPL Version 2**, aber compatibel mit der **GPL Version 3**.
* **OSI (Open Source Initiative):** Powstała wersja Apache 2.0 jako popularna i standardowa licencja Open-Source an.

### 2. Czy darf man damit machen? (Die Freiheiten)
Unter der Apache 2.0 Lizenz darf jeder (auch Firmen):
* **Die Software für jeden Zweck nutzen:** Kommerziell, prywatny, stażysta – wszystkie błędy.
* **Den Code verändern:** Du kannst das Tool anpassen, wie du willst.
* **Die Software kopieren und verbreiten:** Du darfst sie weitergeben.
* **Weryfikacja wersji:** Du darfst Software, die auf Apache-Code basiert, verkaufen.

### 3. Die Besonderheit: „Pozwalający” (Freizügig)
Im Gegensatz zur GPL (General Public License) zawiera Apache-Lizenz **keinen „Copyleft”-Effekt**.
* Das bedeutet: Wenn du Aura (w wersji Apache 2.0) nimmst und veränderst, **musst** du deinen neuen Code nicht zwingend wieder als Open Source veröffentlichen. Du kannst ihn auch in ein geschlossenes, kommerzielles Produkt einbauen.

### 4. Ein großer Vorteil: Die Patentklausel
Die Apache 2.0 Lizenz ist bei Firmen sehr beliebt (często wierzy się w MIT lub BSD), weil sie eine **ausdrückliche Patenterteilung** enthält.
* Jeder, der Code zum Projekt beiträgt, gewährt den Nutzern automatisch eine Lizenz für alle Patente, die er an diesem Code hält.
* Das schützt die Nutzer vor „Patent-Trollen”. Wenn jemand dich wegen Patentverletzung verklagt, der selbst am Projekt mitgearbeitet hat, verliert er automatisch seine Rechte an der Software.

### Zusammenfassung:
W przypadku **Apache 2.0**, najbardziej profesjonalny sygnał:

1. **Rechtssicherheit:** Besonders für große Firmen (wie w branży motoryzacyjnej lub finansów) ist das die „sicherste” Lizenz.
2. **Elastyczność:** Sie können dein Tool problemlos in ihre internen Toolchains integrieren, ohne Angst vor dem Copyleft-Effekt haben zu müssen.

**Fazit:** Jest to FOSS (oprogramowanie bezpłatne i otwarte) w najlepszym wydaniu. Es ist „frei” sowohl im Sinne von „kostenlos” jest także auch im Sinne von „Freiheit”.
