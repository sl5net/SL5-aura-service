# Die License erklärt in deutscher Sprache

La **licence Apache, version 2.0** est définitivement le **logiciel libre** (logiciel gratuit) et également le **logiciel Open Source**.

Ici, vous trouverez les détails, ce qui est vrai et ce qui était pour vous :

### 1. Annonces officielles
* **FSF (Free Software Foundation) :** Vous avez créé Apache 2.0 en tant que licence pour les « logiciels gratuits ». Si ces versions ne sont **pas compatibles avec la GPL version 2**, elles sont également compatibles avec la **GPL version 3**.
* **OSI (Open Source Initiative) :** Apache 2.0 est l'une des licences Open Source les plus populaires et les plus standardisées.

### 2. Darf Man était-il Damit Machen ? (Les Freiheiten)
Sous la licence Apache 2.0, vous devez le faire (au sein de l'entreprise) :
* **Le logiciel pour chacun d'entre eux est disponible :** Kommerziell, privé, stagiaire – tous les employés.
* **Den Code verändern:** Vous pouvez utiliser l'outil, comme vous le souhaitez.
* **Le logiciel a été copié et publié :** Vous êtes prêt à le faire.
* **Sogar verkaufen:** Le logiciel le plus fiable, basé sur Apache-Code, est verkaufen.

### 3. Die Besonderheit : "Permissif" (Freizügig)
Je suis Gegensatz zur GPL (General Public License) avec la licence Apache ** avec effet "Copyleft" **.
* Ceci est vrai : lorsque l'Aura (sous Apache 2.0) est nimmst et veränderst, **doit** que votre nouveau code ne soit pas similaire à celui de l'Open Source. Vous pouvez également entrer dans un produit commercial ou commercial.

### 4. Un plus grand avantage : le Klausel des brevets
La licence Apache 2.0 est une entreprise très croyante (souvent croyante comme le MIT ou BSD), mais elle est également une **entreprise brevetée** certifiée.
* En effet, le code d'un projet est géré automatiquement par une licence pour tous les brevets, qui est le code hält.
* Das schützt die Nutzer vor "Patent-Trollen". Lorsque je demande la publication de brevets, l'auto-projet mitgearbeitet est publié automatiquement dans le cadre du droit du logiciel.

### Fassammenfassung :
Sous **Apache 2.0**, signalez le professionnalisme :

1. **Rechtssicherheit :** Les demandes pour les grandes entreprises (comme dans l'automobile ou la finance) sont la licence "sûre".
2. **Flexibilité :** Si vous avez des problèmes d'outils dans vos chaînes d'outils internes intégrées, sans angoisse pour l'effet Copyleft, vous devez les utiliser.

**Facile :** Il s'agit d'un FOSS (logiciel libre et open source) dans le meilleur des cas. C'est "libre" sowohl im Sinne von "kostenlos" et aussi im sinne von "Freiheit".
