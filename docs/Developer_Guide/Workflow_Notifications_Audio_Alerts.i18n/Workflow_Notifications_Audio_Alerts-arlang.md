# إخطارات سير العمل (التنبيهات الصوتية)

لتحسين الإنتاجية، يمكنك تكوين اسم مستعار Git محلي يدفع التعليمات البرمجية الخاصة بك وينبهك تلقائيًا (عبر الصوت أو الصوت) بمجرد انتهاء سير عمل GitHub Actions. وهذا يمنع "إرهاق مشاهدة GitHub" ويتيح لك التركيز على المهام الأخرى.

                                      ### المتطلبات الأساسية

أنت بحاجة إلى **GitHub CLI** ومحرك تحويل النص إلى كلام أو مشغل الصوت المثبت على نظامك.

                                **بالنسبة إلى Manjaro/Arch Linux:**
```bash
sudo pacman -S github-cli espeak-ng
gh auth login
```

                                                                 ### يثبت

قم بتشغيل الأمر التالي في المحطة الطرفية الخاصة بك لإنشاء اسم مستعار Git عالمي يسمى "Pushsound":

```bash
git config --global alias.pushsound '!git push && sleep 3 && gh run watch $(gh run list --limit 1 --json databaseId --jq ".[0].databaseId") && espeak-ng "all github workflow has finished"'
```

                                                       ### الاستخدام

                 بدلاً من `git Push`، قم ببساطة بتشغيل:
```bash
git pushsound
```
ستنتظر المحطة الطرفية الخاصة بك حتى يكتمل سير العمل ثم تعلن: *"انتهى كل سير عمل جيثب"*.

                                                                          ---

                                          ### التخصيص والبدائل

اعتمادًا على تفضيلاتك، قد ترغب في استخدام اسم مستعار مختلف أو طريقة إعلام مختلفة.

                #### 1. الأسماء المستعارة الموصى بها
إذا كانت كلمة "pushsound" طويلة جدًا بحيث لا يمكن كتابتها، ففكر في هذه البدائل:
* `git pw` (الدفع والمراقبة) — **موصى به للسرعة.**
* ``git sync`` (يعني الضغط وانتظار "الضوء الأخضر")
                                     * ``git Palert`` (تنبيه الدفع)

                                        #### 2. أنماط الإشعارات
يمكنك تبديل الجزء `espeak-ng` بأنواع أخرى من التنبيهات:

                                        * **إخطار سطح المكتب:**
`... && إعلام-إرسال "إجراء GitHub" "انتهى سير العمل!"
                                      * **صوت النظام (الجرس):**
            `... && paplay /usr/share/sounds/freedesktop/stereo/complete.oga`
                            * **المجموعة (الصوت + الصوت):**
`... && paplay /usr/share/sounds/freedesktop/stereo/message.oga && espeak-ng "Done"`

                           #### 3. متقدم: إصدار آمن للفريق
إذا قام العديد من المطورين بالدفع إلى نفس المستودع في وقت واحد، فقد يتتبع الأمر الافتراضي التشغيل الخاطئ. استخدم هذا الإصدار "الآمن للفرع" لمشاهدة فرعك الحالي فقط:

```bash
git config --global alias.pw '!git push && sleep 3 && gh run watch $(gh run list --branch $(git branch --show-current) --limit 1 --json databaseId --jq ".[0].databaseId") && espeak-ng "Workflow finished"'
```

                           ### استكشاف الأخطاء وإصلاحها
* **"لم يتم العثور على عمليات تشغيل":** قمنا بتضمين "النوم 3" لأن GitHub يستغرق بعض الوقت لتسجيل الدفعة وبدء سير العمل. إذا كان لديك اتصال بطيء جدًا، فقد تحتاج إلى زيادة هذا إلى "النوم 5".
* **الصافرات الطرفية:** إذا لم يعمل `espeak-ng`، فتأكد من عدم كتم الصوت ومن تثبيت الحزمة بشكل صحيح.
