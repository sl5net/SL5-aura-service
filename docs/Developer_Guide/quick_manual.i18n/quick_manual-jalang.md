## 高度なルールの属性

標準フィールドに加えて、特別なオプションを使用してルールを拡張できます。

### `only_in_windows` (ウィンドウ タイトル フィルター)
その名前にもかかわらず、この属性は **OS に依存しません**。現在アクティブなウィンドウのタイトルに基づいてルールをフィルタリングします。

* **機能:** ルールは、アクティブなウィンドウのタイトルが提供されたパターン (正規表現) のいずれかに一致する場合にのみ処理されます。
*   **例：**
    ```python
    (
        '|', 
        r'\b(pipe|symbol)\b', 
        75, 
        {'only_in_windows': ['Terminal', 'Console', 'iTerm']}
    ),
    ```
*この場合、置換はユーザーがターミナル ウィンドウ内で作業している場合にのみ行われます。*

### `on_match_exec` (スクリプトの実行)
ルールが一致した場合に外部 Python スクリプトをトリガーできるようにします。

* **構文:** `'on_match_exec': [CONFIG_DIR / 'script.py']`
* **使用例:** API 呼び出し、ファイル システム タスク、動的コンテンツの生成などの複雑なアクションに最適です。
