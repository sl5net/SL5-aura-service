# DEV_MODE सेटअप गाइड

## समस्या

चूंकि हम वेयलैंड के साथ संगत हैं इसलिए हम लॉगिंग के लिए `थ्रेडिंग.लॉक` का उपयोग करते हैं।

अब (21.3.'26 शनि) लॉगिंग के नियम बदल गए हैं। मंज़रो में यह समस्या रहित था।

जब `DEV_MODE = 1` सक्रिय होता है, तो ऑरा प्रति सेकंड सैकड़ों लॉग प्रविष्टियाँ उत्पन्न करता है
अनेक धागों से. इससे `SafeStreamToLogger` में गतिरोध उत्पन्न हो सकता है, जिससे समस्या उत्पन्न हो सकती है
प्रथम श्रुतलेख ट्रिगर के बाद आभा रुक जाती है।

## मूल कारण
`SafeStreamToLogger` stdout लेखन की सुरक्षा के लिए `threading.Lock` का उपयोग करता है।
उच्च लॉग लोड (DEV_MODE) के तहत, लॉक विवाद सिस्टम पर गतिरोध का कारण बनता है
आक्रामक थ्रेड शेड्यूलिंग के साथ (उदाहरण के लिए नए कर्नेल/ग्लिब के साथ कैचीओएस)।
