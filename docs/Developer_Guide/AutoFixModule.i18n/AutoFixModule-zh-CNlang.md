# 自动修复模块（快速规则输入模式）

## 它的作用

当您在地图文件中键入普通单词（不带引号或 Python 语法）时
像“FUZZY_MAP_pre.py”一样，系统会自动将其转换为有效的规则。

这是创建新规则的最快方法——无需记住格式。

＃＃ 例子

您可以将其输入“FUZZY_MAP_pre.py”：

__代码_块_0__

自动修复模块检测到“NameError”（裸词，无效的 Python）
并自动将文件转换为：

__代码_块_1__

现在将规则编辑为您实际需要的内容：

__代码_块_2__

## 它是如何工作的

自动触发模块 `scripts/py/func/auto_fix_module.py`
当 Aura 在加载地图文件时检测到“NameError”时。

然后它：
1.添加正确的文件路径头
2. 如果缺少的话添加“import re”
3. 添加 `FUZZY_MAP_pre = [` 列表定义
4. 将裸词转换为 `('word', 'word'),` 元组
5. 使用`]`关闭列表

## 规则和限制

- 仅适用于小于 **1KB** 的文件（安全限制）
- 仅适用于：`FUZZY_MAP.py`、`FUZZY_MAP_pre.py`、`PUNCTUATION_MAP.py`
- 文件必须位于有效的语言文件夹中（例如“de-DE/”）
- 一次适用于多个单词（例如来自电话簿列表）

## 已知问题（未完全测试）

> ⚠️ 该模块可以正常运行，但尚未经过详尽的测试。以下情况可能无法正常工作：

- **数字** – `5` 或 `6` 不是有效的 Python 标识符，自动修复可能无法处理它们
- **特殊字符** – 带有“-”、“.”、变音符号的单词可能不会触发“NameError”
- **多字条目** – `thomas mueller`（带空格）导致 `SyntaxError` 而不是 `NameError`，因此可能不会触发自动修复
- **逗号分隔值** – `drei, vier` 可以按原样插入，而不会成为正确的元组

如果自动修复未触发，请手动添加规则：

此注释会自动添加，作为规则方向的提醒：

对于“PUNCTUATION_MAP.py”，方向相反：“# from->too”
