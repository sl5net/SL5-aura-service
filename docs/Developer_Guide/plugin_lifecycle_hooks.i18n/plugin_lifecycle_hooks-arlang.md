# خطافات دورة حياة البرنامج المساعد

يدعم Aura SL5 خطافات دورة الحياة التي تسمح للمكونات الإضافية (الخرائط) بتنفيذ منطق محدد تلقائيًا عندما تتغير حالتها.

                                                ## الخطاف `on_reload()`

تعد وظيفة `on_reload()` وظيفة اختيارية خاصة يمكنك تعريفها داخل أي خريطة مكون إضافي (`.py`).

                                                                 ### سلوك
* **المشغل:** يتم تنفيذ هذه الوظيفة **مباشرة بعد** إعادة تحميل الوحدة بنجاح (تغيير الملف المكتشف + المشغل الصوتي).
* **السياق:** يتم تشغيله ضمن تدفق التطبيق الرئيسي.
* **النطاق:** لا يتم تنفيذه أثناء بدء التشغيل الأولي للنظام (البدء البارد). إنه مخصص فقط لسيناريوهات * إعادة * التحميل.

                                            ### حالات الاستخدام
* **الأمان:** إعادة تشفير الملفات الحساسة أو إعادة ضغطها تلقائيًا بعد التحرير.
* **إدارة الحالة:** إعادة تعيين العدادات العامة أو مسح ذاكرات تخزين مؤقت محددة.
* **الإشعار:** تسجيل معلومات تصحيح محددة للتحقق من تطبيق التغيير.

                           ### التفاصيل الفنية والسلامة
* **معالجة الأخطاء:** يتم تغليف التنفيذ في كتلة `محاولة/باستثناء`. إذا تعطلت وظيفة `on_reload` (على سبيل المثال، `DivisionByZero`)، فسيتم تسجيل خطأ (`❌ خطأ في تنفيذ on_reload...`) ولكن **لن يتعطل Aura**.
* **الأداء:** تعمل الوظيفة بشكل متزامن. تجنب المهام طويلة الأمد (مثل التنزيلات الكبيرة) مباشرة في هذه الوظيفة، لأنها ستمنع معالجة الأوامر الصوتية لفترة وجيزة. بالنسبة للمهام الثقيلة، قم بإنشاء خيط.
