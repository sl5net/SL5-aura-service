# 플러그인 수명주기 후크

Aura SL5는 상태가 변경될 때 플러그인(지도)이 특정 로직을 자동으로 실행할 수 있도록 하는 수명 주기 후크를 지원합니다.

## `on_reload()` 후크

`on_reload()` 함수는 플러그인 맵(`.py`) 내에서 정의할 수 있는 특별한 선택 함수입니다.

### 행동
* **트리거:** 이 기능은 모듈이 성공적으로 핫 리로드된 **즉시** 실행됩니다(파일 변경 감지 + 음성 트리거).
* **컨텍스트:** 기본 애플리케이션 흐름 내에서 실행됩니다.
* **범위:** 초기 시스템 시작(콜드 스타트) 중에는 실행되지 않습니다**. 이는 *다시* 로드하는 시나리오에만 적용됩니다.

### 사용 사례
* **보안:** 편집 후 중요한 파일을 자동으로 다시 암호화하거나 다시 압축합니다.
* **상태 관리:** 전역 카운터를 재설정하거나 특정 캐시를 지웁니다.
* **알림:** 변경 사항이 적용되었는지 확인하기 위해 특정 디버그 정보를 기록합니다.

### 기술 세부정보 및 안전
* **오류 처리:** 실행은 `try/ex` 블록으로 래핑됩니다. `on_reload` 함수가 충돌하는 경우(예: `DivisionByZero`) 오류(`❌ On_reload 실행 중 오류 발생...`)를 기록하지만 **Aura는 충돌하지 않습니다**.
* **성능:** 함수가 동기적으로 실행됩니다. 음성 명령 처리가 잠시 차단되므로 이 기능에서 직접 장기 실행 작업(예: 대용량 다운로드)을 피하세요. 무거운 작업의 경우 스레드를 생성합니다.
