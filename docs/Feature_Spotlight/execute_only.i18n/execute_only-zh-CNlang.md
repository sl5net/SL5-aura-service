# 规则属性：`execute_only`（实验性，26 年 7 月 7 日星期二）

`execute_only` 属性是一个实验性配置选项，专为仅触发外部脚本而不修改或替换输入文本的规则而设计。

＃＃ 概述
- **类型：** `bool` （例如，`True` 或 `False`）
- **主要用例：** 通常与 `on_match_exec` 结合使用来运行外部脚本。

## 它是如何工作的以及当前的行为
- **速度优化：**（仅几毫秒）绕过文本后处理和文本替换例程，加快触发操作的立即执行速度。
- **无排除/失败副作用：** 将 `execute_only` 设置为 `True` 并不会阻止其他匹配规则评估相同的输入文本。
- **停止流程：** 如果您需要停止后续规则处理相同的输入文本，您当前必须手动终止执行流程（例如，通过在触发的脚本或规则集处理程序末尾引发异常）。
