# ऑटो-ज़िप वर्कफ़्लो और एंबेडेड दस्तावेज़ीकरण

## अवधारणा
SL5 ऑरा स्वचालित रूप से `_` से शुरू होने वाले फ़ोल्डरों की निगरानी करता है (उदाहरण के लिए, `_my_application`)। जब परिवर्तनों का पता चलता है, तो ऑरा स्वचालित रूप से फ़ोल्डर को एक ज़िप संग्रह में संपीड़ित कर देता है।

**महत्वपूर्ण बाधा:**
ऑरा का "हॉट-रीलोड" और मॉनिटरिंग सिस्टम विशेष रूप से **मान्य पायथन फाइलों** में बदलावों को सुनता है। एक साधारण टेक्स्ट फ़ाइल (`.txt`) अपडेट ऑटो-ज़िप प्रक्रिया को **नहीं** ट्रिगर करेगा।

## "एम्बेडेड डॉक्स" पैटर्न
यह सुनिश्चित करते हुए कि ऑरा परिवर्तन का पता लगाता है और ज़िप को अपडेट करता है, गैर-तकनीकी प्राप्तकर्ताओं (जैसे, एचआर, क्लाइंट) के लिए निर्देश शामिल करने के लिए, हम **पायथन डॉकस्ट्रिंग फ़ाइल** का उपयोग करते हैं।

यह फ़ाइल तकनीकी रूप से एक वैध पायथन स्क्रिप्ट है (ऑरा के पार्सर को संतुष्ट करती है) लेकिन उपयोगकर्ता के लिए दृश्यमान रूप से एक मानक टेक्स्ट दस्तावेज़ के रूप में दिखाई देती है।

### कार्यान्वयन
अपने मॉनिटर किए गए फ़ोल्डर के अंदर `README_AUTOZIP.py` नाम की एक फ़ाइल बनाएं।

**स्टाइल गाइड:**
1. स्वागत योग्य होने के लिए पहली पंक्ति के रूप में `# दस्तावेज़ीकरण` का उपयोग करें (तकनीकी स्क्रिप्ट नाम के बजाय)।
2. सामग्री के लिए ट्रिपल-कोट ('''''`) डॉकस्ट्रिंग का उपयोग करें।
3. किसी अन्य कोड की आवश्यकता नहीं है.
