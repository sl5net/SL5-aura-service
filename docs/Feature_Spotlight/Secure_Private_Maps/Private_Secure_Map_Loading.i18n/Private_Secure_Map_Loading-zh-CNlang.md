# 特色亮点：安全的私人地图加载和自动打包

本文档概述了管理敏感地图插件（例如客户端数据、专有命令）的架构，允许**实时编辑**，同时执行**安全最佳实践**以防止意外的 Git 暴露。

---

## 1. 概念：“Matryoshka”安全

为了在使用标准工具时确保最大程度的隐私，Aura 使用 **Matryoshka（俄罗斯娃娃）** 嵌套策略来加密档案。

1. **外层：** 使用 **AES-256** 加密的标准 ZIP 文件（通过系统 `zip` 命令）。
* *外观：* 仅包含 **一个** 名为“aura_secure.blob”的文件。
* *好处：* 隐藏文件名和目录结构以防窥探。
2. **内层（Blob）：** Blob 内未加密的 ZIP 容器。
* *内容：*实际的目录结构和Python文件。
3. **工作状态：** 解锁后，文件将被提取到以下划线前缀的临时文件夹中（例如“_private”）。
* *安全性：* 该文件夹被 `.gitignore` 严格忽略。

---

## 2. 技术工作流程

### A. 安全门（启动）
在解压任何内容之前，Aura 会检查“scripts/py/func/map_reloader.py”是否有特定的“.gitignore”规则。
* **规则 1:** `config/maps/**/.*` （保护关键文件）
* **规则 2:** `config/maps/**/_*` （保护工作目录）
如果缺少这些，系统**中止**。

### B. 解包（异常驱动）
1. 用户创建一个包含密码（以明文或注释形式）的密钥文件（例如“.auth_key.py”）。
2. Aura 检测此文件和相应的 ZIP（例如“private.zip”）。
3. Aura 使用密钥解密外部 ZIP。
4. Aura 检测 `aura_secure.blob`，提取内层，并将文件移动到工作目录 `_private`。

### C. 实时编辑和自动打包（循环）
这就是系统变得“自我修复”的地方：

1. **编辑：** 您修改 `_private/` 中的文件并保存。
2. **触发：** Aura 检测到变化并重新加载模块。
3. **Lifecycle Hook：** 模块触发其`on_reload()`函数。
4. **SecurePacker：** 私有文件夹根目录中的脚本（`secure_packer.py`）执行：
* 它创建内部 ZIP（结构）。
* 它将其重命名为“.blob”。
* 它调用系统“zip”命令，使用“.key”文件中的密码将其加密到外部存档中。

**结果：** 您的“private.zip”始终与最新的更改保持同步，但 Git 只能看到二进制 ZIP 文件的更改。

# 开发人员指南：插件生命周期挂钩

Aura SL5 允许插件（地图）定义特定的“钩子”，当模块状态发生变化时自动执行。这对于**安全私人地图**系统等高级工作流程至关重要。

## `on_reload()` 钩子

`on_reload()` 函数是一个可选函数，您可以在任何 Map 模块中定义。

＃＃＃ 行为
* **触发：**模块成功**热重载**后立即执行（文件修改+语音触发）。
* **上下文：** 在主应用程序线程内运行。
* **安全：** 包裹在 `try/ except` 块中。此处的错误将被记录，但不会导致应用程序崩溃。

### 最佳实践
1. **保持快速：** 不要在主钩子中运行长时间阻塞任务（例如大量下载）。如有必要，请使用线程。
2. **幂等性：** 确保您的钩子可以多次运行而不破坏事物（例如，不要无休止地附加到文件，而是重写它）。
