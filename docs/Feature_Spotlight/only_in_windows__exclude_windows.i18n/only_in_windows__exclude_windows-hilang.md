# नियम विशेषताएँ: `only_in_windows` और `exclude_windows`

ये दो विशेषताएँ नियंत्रित करती हैं **जिसमें किसी नियम को सक्रिय विंडो में सक्रिय करने की अनुमति है**।
वे एक नियम के `विकल्प` निर्देश के अंदर परिभाषित होते हैं और **रेगेक्स पैटर्न की सूची** को स्वीकार करते हैं
जो वर्तमान सक्रिय विंडो शीर्षक (`_active_window_title`) से मेल खाते हैं।

---

## `केवल_विंडोज़ में`

नियम **केवल तभी सक्रिय होता है जब** सक्रिय विंडो शीर्षक दिए गए पैटर्न में से **कम से कम एक** से मेल खाता हो।
अन्य सभी विंडो को नजरअंदाज कर दिया जाता है।

**उपयोग मामला:** किसी नियम को किसी विशिष्ट एप्लिकेशन तक सीमित रखें।


> नियम **केवल** तभी सक्रिय होगा जब फ़ायरफ़ॉक्स या क्रोमियम सक्रिय विंडो हो।

---

## `बहिष्कृत_विंडोज़`

नियम तब तक प्रभावी होता है जब तक सक्रिय विंडो शीर्षक दिए गए पैटर्न में से कम से कम एक से मेल नहीं खाता।
मिलान वाली विंडो छोड़ दी गई हैं.

**उपयोग मामला:** विशिष्ट अनुप्रयोगों के लिए एक नियम अक्षम करें।

उदाहरण

__CODE_ब्लॉक_0__



मिलान **केस-असंवेदनशील** है और पायथन **नियमित अभिव्यक्तियों** का उपयोग करता है।
