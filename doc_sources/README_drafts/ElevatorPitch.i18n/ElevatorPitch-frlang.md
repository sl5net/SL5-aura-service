## Éléments minimalistes pour l'outil de gestion

### Option 1 : Fokus auf die Architektur (Die Stärke)

> **Service SL5 Aura : configuration sans assistance hors ligne.**
>
> * **100 % hors ligne :** Vosk STT et Lokaler LanguageTool-Server.
> * **Kontrollierte Verarbeitung :** Sequenzielle, kaskadierende Regelausführung (Fuzzy Maps).
> * **Commandos :** Priorité activée par match complet (`^$`) Stopp-Kriterium.
> * **Anpassung:** Configuration (Regel-Tupel) ersetzt Programmierung.
> * **Erweiterbar:** Structure de plug-in simple (pas de priorité).

### Option 2 : Fokus auf den Nutzen (Was es tut)

> **SL5 Aura : Le Diktier- und Kommandosteuerung privé.**
>
> * Sprachbefehle und Diktat – 100% privé, sans Cloud.
> * Intelligente Korrektur : Grammatik (LT) + Custom-Regelwerk.
> * Transformation de texte flexible (Code-Konventionen, Abkürzungen).
> * Barre de contrôle : une simple règle `RegEx` définit une logique complexe.

## Explications minimalistes pour l'ensemble de l'outil

### Option 1 : Focus sur l'architecture et le contrôle (l'USP technique)

> **Service SL5 Aura : assistant vocal hors ligne configurable.**
>
> * **100 % hors ligne :** Serveur Vosk STT et Local LanguageTool.
> * **Traitement contrôlé :** Exécution séquentielle de règles en cascade (Fuzzy Maps).
> * **Commandes :** Priorité stricte via le critère d'arrêt Full Match ($`^$`).
> * **Personnalisation :** La configuration (Tuples de règles) remplace le codage traditionnel.
> * **Extensible :** Structure de plug-in simple (faible priorité via l'ordre des fichiers).

### Option 2 : Se concentrer sur la fonctionnalité et la confidentialité (l'avantage pour l'utilisateur)

> **SL5 Aura : dictée privée et contrôle des commandes.**
>
> * Commandes vocales et dictée – 100 % privées, pas de cloud.
> * Correction intelligente : Grammaire (LT) + Moteur de règles personnalisées.
> * Transformation de texte flexible (conventions de code, abréviations).
> * Entièrement contrôlé : des règles RegEx simples définissent une logique complexe.
