# STT 프로젝트 테스트를 위한 VirtualBox 설정 가이드

이 가이드는 VirtualBox에서 안정적이고 성능이 뛰어난 Ubuntu 24.04 가상 머신을 설정하기 위한 권장 단계를 제공합니다. 이 지침을 따르면 STT 애플리케이션을 테스트하기 위한 일관된 환경이 조성되고 설치 속도 저하, 시스템 정지, 클립보드 기능 누락과 같은 일반적인 문제를 방지할 수 있습니다.

## 전제조건

- 호스트 컴퓨터에 VirtualBox가 설치되어 있습니다.
- Ubuntu 24.04 Desktop ISO 파일이 다운로드되었습니다.

## 참조 호스트 하드웨어

이 구성은 다음 호스트 시스템에서 테스트되고 검증되었습니다. 성능은 하드웨어에 따라 다를 수 있지만 안정성 설정은 보편적으로 적용되어야 합니다.

- **운영 체제:** Manjaro Linux
- **커널:** 6.6.94
- **프로세서:** 16 × AMD Ryzen 7 3700X
- **메모리:** 31.3GiB RAM
- **그래픽 프로세서:** NVIDIA GeForce GTX 1050 Ti

---

## 1부: VM 생성 및 구성

이러한 설정은 성능과 안정성에 매우 중요합니다.

### 1.1단계: 새 가상 머신 생성

1. VirtualBox에서 **새로 만들기**를 클릭합니다.
2. **이름:** `Ubuntu STT Tester`(또는 유사).
3. **ISO 이미지:** 이 필드를 비워 두세요.
4. **"무인 설치 건너뛰기"** 확인란을 선택합니다.
5. **다음**을 클릭합니다.
6. **하드웨어:**
- **기본 메모리:** `4096MB` 이상.
- **프로세서:** `4` 이상.
7. **다음**을 클릭합니다.

### 1.2단계: 가상 하드 디스크 생성(중요)

이는 빠른 설치와 성능을 위한 가장 중요한 단계입니다.

1. **"지금 가상 하드 디스크 만들기"**를 선택합니다.
2. 디스크 크기를 **40GB** 이상으로 설정합니다.
3. 다음 화면에서 저장 유형을 **"고정 크기"**로 변경합니다.
> **이유?** 고정 크기 디스크는 사전 할당되어 "동적 할당" 디스크가 설치 중에 지속적으로 크기가 조정될 때 발생하는 대규모 I/O 병목 현상을 방지합니다.
4. **만들기**를 클릭하고 프로세스가 완료될 때까지 기다립니다.

### 1.3단계: 최종 VM 설정

새로 생성된 VM을 선택하고 **설정**을 클릭합니다. 다음을 구성합니다.

- **시스템 -> 마더보드:**
- **칩셋:** `ICH9`
- **"EFI 활성화(특수 OS에만 해당)"**를 선택합니다.

- **디스플레이 -> 화면:**
- **그래픽 컨트롤러:** `VMSVGA`
- **"3D 가속 활성화"를 선택 취소합니다**.
> **이유는 무엇입니까?** 3D 가속은 Linux 게스트에서 시스템이 중단되고 정지되는 일반적인 원인입니다. 비활성화하면 안정성이 크게 향상됩니다.

-   **저장:**
- **SATA 컨트롤러**를 선택합니다. **"호스트 I/O 캐시 사용"** 확인란을 선택합니다.
- 가상 디스크 파일(`.vdi`)을 선택합니다. **"솔리드 스테이트 드라이브"** 확인란을 선택합니다.
- **빈** 광학 드라이브를 선택합니다. 오른쪽에 있는 CD 아이콘을 클릭하고 **"디스크 파일 선택..."**을 클릭하여 Ubuntu 24.04 ISO를 연결하세요.

모든 설정을 저장하려면 **확인**을 클릭하세요.

## 2부: 우분투 OS 설치

1. VM을 시작합니다.
2. 언어 및 키보드 설정을 진행하세요.
3. "업데이트 및 기타 소프트웨어"에 도달하면 다음을 선택합니다.
- **최소 설치**.
- **선택 취소** "Ubuntu 설치 중 업데이트 다운로드".
4. 설치가 완료될 때까지 설치를 계속합니다.
5. 완료되면 VM을 재부팅합니다. 프롬프트에서 설치 매체를 제거하십시오(Enter를 누르십시오).

---

## 3부: 설치 후(게스트 추가)

이 단계에서는 클립보드 공유, 끌어서 놓기 및 자동 화면 크기 조정이 가능합니다.

### 3.1단계: 호스트에 게스트 추가 ISO 설치(필요한 경우)

**호스트 머신**에 Guest Additions ISO 패키지가 설치되어 있는지 확인하세요.

- **아치에 / 만자로:**
    ```bash
    sudo pacman -S virtualbox-guest-iso
    ```
- **데비안/우분투:**
    ```bash
    sudo apt install virtualbox-guest-additions-iso
    ```

### 3.2단계: Ubuntu VM 내에 게스트 추가 설치

**실행 중인 Ubuntu VM** 내에서 다음 단계를 수행하세요.

1. **Ubuntu 준비:** 터미널을 열고 다음 명령을 실행하여 빌드 종속성을 설치합니다.
    ```bash
    sudo apt update
    sudo apt install build-essential dkms linux-headers-$(uname -r)
    ```
2. **CD 삽입:** VirtualBox 상단 메뉴에서 **장치 -> 게스트 추가 CD 이미지 삽입...**으로 이동합니다.
3. **설치 프로그램 실행:**
- 소프트웨어 실행을 요청하는 대화 상자가 나타날 수 있습니다. **실행**을 클릭합니다.
- 대화 상자가 나타나지 않으면 파일 관리자를 열고 `VBox_GAs...` CD를 마우스 오른쪽 버튼으로 클릭한 후 **"터미널에서 열기"**를 선택하고 다음 명령을 실행합니다.
      ```bash
      sudo ./VBoxLinuxAdditions.run
      ```
4. **재부팅:** 설치가 완료된 후 VM을 재부팅합니다.
    ```bash
    reboot
    ```
5. **기능 활성화:** 재부팅 후 **장치** 메뉴로 이동하여 **공유 클립보드 -> 양방향** 및 **끌어서 놓기 -> 양방향**을 활성화합니다.

이제 안정적이고 성능이 뛰어난 테스트 환경이 준비되었습니다.
