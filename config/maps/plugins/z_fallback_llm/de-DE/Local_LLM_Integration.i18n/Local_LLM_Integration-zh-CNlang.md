# 🧠 SL5 Aura 混合模式：本地 LLM 和剪贴板集成

**状态：**稳定
**技术：** Ollama (Llama 3.2) + 文件桥架构
**隐私：** 100% 离线

## 概念：“架构师和实习生”

传统上，Aura 依赖于确定性规则 (RegEx) – 快速且精确。这是**“建筑师”**。
**本地 LLM 插件**充当**“实习生”**：它处理模糊请求、总结文本并回答一般问题。

## 🛠 架构：剪贴板桥

由于Linux（Wayland/X11）中的安全限制，后台进程（如Aura）通常无法直接访问剪贴板。我们通过 **Bridge Architecture** 解决了这个问题：

1. **提供程序（用户会话）：** 一个小型 shell 脚本 (`clipboard_bridge.sh`) 在用户会话中运行。它监视剪贴板并将其内容镜像到临时文件（`/tmp/aura_clipboard.txt`）。
2. **消费者（Aura）：** Python插件读取此文件。由于文件访问是通用的，因此可以绕过权限问题。

* **提示工程：** 将当前剪贴板数据优先于历史对话上下文，以防止出现幻觉。
